# ❓ Frequently Asked C Theory Questions

Interviewers often ask conceptual questions before coding.

### 1. What is C Programming?

A general-purpose procedural programming language developed by Dennis Ritchie in 1972.

---

### 2. What is a Pointer?

A pointer is a variable that stores the memory address of another variable.

---

### 3. Difference Between Structure and Union

| Structure | Union |
|----------|-------|
| Separate memory for each member | Shared memory |
| Larger memory usage | Smaller memory usage |
| Members can store values simultaneously | Only one member is active at a time |

---

### 4. Difference Between malloc() and calloc()

| malloc() | calloc() |
|----------|-----------|
| Allocates memory | Allocates and initializes memory |
| Garbage values | Zero-initialized |

---

### 5. What is Recursion?

A function calling itself until a base condition is reached.

---

### 6. What is the Difference Between Call by Value and Call by Reference?

- Call by Value passes a copy of the variable.
- Call by Reference passes the memory address using pointers.

---

### 7. What is a Dangling Pointer?

A pointer that refers to memory that has already been freed.

---

### 8. What is a NULL Pointer?

A pointer that points to no valid memory location.

---

### 9. Why is `main()` Important?

It is the entry point of every C program.

---

### 10. What is the Difference Between `=` and `==`?

- `=` assigns a value.
- `==` compares two values.

# ⚠ Common Mistakes During Interviews

Avoid these common errors:

- Forgetting to initialize variables.
- Missing semicolons.
- Incorrect loop conditions.
- Using `=` instead of `==`.
- Forgetting to free dynamically allocated memory.
- Not handling edge cases.
- Ignoring array bounds.
- Forgetting the null terminator (`'\0'`) in strings.

# 🎤 Technical Interview Tips

During the interview:

- Read the question carefully.
- Clarify doubts before writing code.
- Start with a simple solution.
- Explain your approach.
- Mention time complexity.
- Mention space complexity.
- Test your solution using sample inputs.
- Stay calm if you make mistakes.
